Internationalization Handling

Every thoughtfully written article deserves to be seen by more people, to transcend language barriers, and to be read by audiences worldwide. Kecare provides you with powerful internationalization (i18n) support, making it easy for your content to reach a global audience.

Why Internationalization is Needed

Expanding Audience Reach

  • Reach readers from different language backgrounds
  • Enhance the article's international influence
  • Increase blog traffic

Enhance User Experience

  • Readers can read in their native language
  • Reduces comprehension difficulties caused by language barriers
  • Provides a more user-friendly reading experience

SEO Optimization

  • Multilingual content helps with search engine indexing
  • Improves ranking in search results across different regions

Quick Start

Step 1: Enable Translation in the Article

Add a translate field to the article's Front Matter:

---
title: 我的第一篇文章
translate: ['zh-CN', 'en-US', 'ja-JP']
---

⚠️ Important: The first language code must be the original language of the article.

Step 2: Create Configuration File

Create a kecare.config.ts file in the .kecare directory.

Step 3: Configure Translation Parameters

// .kecare/kecare.config.ts
export async function kecareConfig() {
    return {
        llm: {
            model: 'deepseek-chat',
            apiKey: 'your-api-key',
            apiBaseUrl: 'https://api.deepseek.com/v1',
            prompt: ''  // 可选：额外的翻译提示词
        }
    };
}

Step 4: Run the Generator

kecare gen .

Kecare automatically translates articles and generates corresponding language versions.

Configuring Translation Features

Configuration Item Description

Field Type Required Description
model string Yes LLM model name
apiKey string Yes API key
apiBaseUrl string Yes API service address
prompt string No Custom translation prompt

Built-in System Prompts

Kecare comes with optimized translation prompts:

你是一个很好用的翻译助手，请帮我将下面的文章翻译成目标语言，
注意：
- 不要增删任何 HTML 标签/属性和 Markdown 标签/属性
- 如果是代码块的内容，请不要翻译，只翻译文本节点
- 如果是代码内容，请不要翻译，原样放回
- 保留链接 href、图片 src、id、class 不动
- 代码块占位符（如 __CODE_BLOCK_0__）不要翻译，原样保留

Custom Prompts

You can add additional translation instructions via the prompt field:

export async function kecareConfig() {
    return {
        llm: {
            model: 'deepseek-chat',
            apiKey: 'your-api-key',
            apiBaseUrl: 'https://api.deepseek.com/v1',
            prompt: '请使用正式、专业的语气进行翻译。'
        }
    };
}

Language Code Format

Language codes follow the BCP 47 standard:

语言代码-国家/地区代码

For example:

  • zh-CN: Chinese-China
  • en-US: English-United States
  • ja-JP: Japanese-Japan

Token Optimization Strategies

The token cost for AI translation can be high. Kecare provides multiple optimization measures to help you save costs:

Segment-by-Segment Translation

Intelligently segment Markdown articles and submit them to AI for translation paragraph by paragraph:

文章 → 段落1 → 翻译 → 结果1
      → 段落2 → 翻译 → 结果2
      → 段落3 → 翻译 → 结果3

Advantages:

  • Reduce the number of tokens per request
  • Lower the risk of API call failures
  • Facilitate parallel processing

Caching Mechanism

Translation results are automatically cached in the .kecare/.tmp directory:

.kecare/
└── .tmp/
    ├── en-US/
    │   └── abc123/
    │       └── translations.json
    └── ja-JP/
        └── abc123/
            └── translations.json

Advantages:

  • No need to translate the same content repeatedly
  • Reduces the number of API calls

Incremental Update

The system automatically detects changes during the next build:

检测文章变化 → 识别修改段落 → 仅翻译变更部分 → 合并结果

Advantages:

  • Only translate modified content
  • Significantly reduce Token consumption
  • Improve build efficiency

Frequently Asked Questions

Q: What should I do if the translation quality is not satisfactory?

A: You can try the following methods:

  • Add translation style requirements in the prompt
  • Switch to a more powerful model
  • Conduct manual proofreading for key paragraphs

Q: Which translation services are supported?

A: Kecare supports any service compatible with the OpenAI API format

Q: Where are the translation results stored?

A: The translation results are cached in the .kecare/.tmp directory. It is recommended to ignore this directory in .gitignore.

Q: How to clear the translation cache?

A: Simply delete the .kecare/.tmp directory:

rm -rf .kecare/.tmp
