Using Kecare on an Existing Project

If you want to use Kecare to add documentation/usage instructions, etc., to your existing project, you don't need to go through a lot of complex and tedious configuration. Using Kecare, you can conveniently and quickly generate the required documentation pages.

This document will guide you step-by-step on how to integrate Kecare into your existing project, making it easy to get started even for beginners.

Prerequisites

Before you begin, make sure your project meets the following conditions:

Requirement Description
Package Manager npm, pnpm, yarn, or bun is installed
Framework Support Supports Vue, React, Svelte, or traditional server-side rendering frameworks
TypeScript TypeScript is recommended for better type hints

Core Concepts

Before starting the configuration, let's first understand a few core concepts:

What is Kecare?

Kecare is a blog/documentation generator that can:

  1. Parse Markdown Articles: Parse the Markdown files you write into structured data.
  2. Support Internationalization: Automatically translate articles into multiple languages.
  3. Generate Pages: Generate corresponding page files based on the templates you define.

Workflow

Markdown 文章 → Kecare 解析 → 生成器处理 → Vue/React 组件

Key File Descriptions

File Name Purpose Required
articles/ Folder for storing Markdown articles ✅ Yes
*.article.ts Article detail page generator ✅ Yes
*.list.ts Article list page generator ✅ Yes
kecare.config.ts Kecare configuration file (translation, etc.) ⚪ Optional
menus/ Menu configuration folder ⚪ Optional

Quick Start

Step 1: Create the .kecare folder

Create the .kecare folder in your project root directory:

your-project/
├── .kecare/
│   ├── articles/           # 存放文章
│   ├── foo-bar.article.ts  # 文章详情页生成器
│   └── foo-bar.list.ts     # 文章列表页生成器
├── package.json
└── ...

Step 2: Install Kecare

npm install kecare
# 或
pnpm add kecare
# 或
bun add kecare

Step 3: Create the Article Detail Page Generator

Create the foo-bar.article.ts file (the filename can be customized):

// .kecare/foo-bar.article.ts
import { join } from 'node:path';
import type { ArticleVariant, KecareContext } from "kecare";

export const type = 'article-detail';

export async function generator(context: KecareContext, article: ArticleVariant) {
    return {
        // 文章详情页面的 URL 路径
        urlPath: ['articles', article.lang, article.hash].join('/'),
        
        // 生成的文件路径
        fsPath: join(context.projectPath, 'app', 'pages', 'articles', article.lang, `${article.hash}.vue`),
        
        // 页面模板
        template: `
            <script setup lang="ts">
            import ArticleTheme from '~/components/ArticleTheme.vue'
            const article = ${JSON.stringify(article)}
            </script>
            <template>
                <ArticleTheme :article="article" />
            </template>
        `
    };
}

Step 4: Create Article List Page Generator

Create the foo-bar.list.ts file (the filename can be customized):

// .kecare/foo-bar.list.ts
import { join } from 'node:path';
import type { ArticlesRecord, KecareContext } from "kecare";

export const type = 'article-list';

const ARTICLES_PER_PAGE = 5;  // 每页文章数
const TARGET_LANGUAGE = 'zh-CN';  // 目标语言

export function generator(context: KecareContext, articles: ArticlesRecord) {
    const files: Array<{ fsPath: string; template: string }> = [];
    const zhArticles: Array<unknown> = [];

    // 筛选目标语言的文章
    for (const articleHash in articles) {
        const articleLanguages = articles[articleHash]!;
        if (articleLanguages[TARGET_LANGUAGE]) {
            zhArticles.push(articleLanguages[TARGET_LANGUAGE]);
        }
    }

    const totalArticles = zhArticles.length;
    const totalPages = Math.ceil(zhArticles.length / ARTICLES_PER_PAGE);

    // 生成分页页面
    for (let pageIndex = 0; pageIndex < totalPages; pageIndex++) {
        const startIndex = pageIndex * ARTICLES_PER_PAGE;
        const endIndex = startIndex + ARTICLES_PER_PAGE;
        const pageArticles = zhArticles.slice(startIndex, endIndex);
        const pageNumber = pageIndex + 1;
        const fileName = pageNumber === 1 ? 'index.vue' : `page-${pageNumber}.vue`;

        files.push({
            fsPath: join(context.projectPath, 'app', 'pages', fileName),
            template: `
                <script setup lang="ts">
                import BlogLanding from "~/components/BlogLanding.vue";
                const articles = ${JSON.stringify(pageArticles, null, 2)}
                const currentPage = ${pageNumber}
                const totalPages = ${totalPages}
                const totalArticles = ${totalArticles}
                </script>
                <template>
                    <BlogLanding 
                        :articles="articles" 
                        :current-page="currentPage" 
                        :total-pages="totalPages" 
                        :total-articles="totalArticles" 
                    />
                </template>
            `,
        });
    }

    return files;
}

Step 5: Create the Article Display Component

Create a component for displaying articles:

<!-- components/ArticleTheme.vue -->
<script setup lang="ts">
import { useKecareSDK } from 'kecare';
import type { ArticleVariant } from "kecare";

// 初始化 Kecare SDK（用于 Markdown 扩展功能）
const kecareSDK = await useKecareSDK();

onMounted(async () => {
    await nextTick();
    await kecareSDK!.mounted(props.article.rawMarkdown);
});

const props = defineProps<{
    article: ArticleVariant;
}>();
</script>

<template>
    <div class="article-container">
        <h1 class="article-title">{{ props.article.title }}</h1>
        <p class="article-desc">{{ props.article.desc }}</p>
        <div class="article-content" v-html="props.article.html"></div>
    </div>
</template>

Step 6: Create the Article List Component

<!-- components/BlogLanding.vue -->
<script lang="ts" setup>
import type { ArticlesRecord } from "kecare";

const props = defineProps<{
    articles: ArticlesRecord;
    currentPage?: number;
    totalPages?: number;
    totalArticles?: number;
}>();

// 定义文章卡片类型
interface ArticleCard {
    id: string;
    title: string;
    desc: string;
    date?: string;
    author?: string;
    to: string;
    lang: string;
    cover?: string;
}

// 处理文章数据
const articleCards = computed<ArticleCard[]>(() => {
    const records = props.articles ?? [];
    
    if (Array.isArray(records)) {
        return records.map((article: any) => ({
            id: article.hash ?? article.id ?? "",
            title: article.title ?? "",
            desc: article.desc ?? "",
            date: article.date,
            author: article.author,
            to: article.urlPath ?? "#",
            lang: article.lang ?? "",
            cover: article.cover,
        }));
    }
    return [];
});
</script>

<template>
    <div class="blog-container">
        <!-- 文章列表 -->
        <div class="articles">
            <NuxtLink 
                class="article-card" 
                v-for="(article, index) in articleCards" 
                :key="index" 
                :to="article.to"
            >
                <img v-if="article.cover" class="cover" :src="article.cover" :alt="article.title" />
                <h2 class="title">{{ article.title }}</h2>
                <p class="desc">{{ article.desc }}</p>
                <div class="meta">
                    <span>作者: {{ article.author }}</span>
                    <span>{{ article.date }}</span>
                </div>
            </NuxtLink>
        </div>

        <!-- 分页导航 -->
        <nav class="pagination" v-if="(totalPages ?? 1) > 1">
            <NuxtLink 
                v-if="(currentPage ?? 1) > 1" 
                :to="(currentPage ?? 1) === 2 ? '/' : `/page-${(currentPage ?? 1) - 1}`"
            >
                上一页
            </NuxtLink>
            <span>第 {{ currentPage }} / {{ totalPages }} 页</span>
            <NuxtLink 
                v-if="(currentPage ?? 1) < (totalPages ?? 1)" 
                :to="`/page-${(currentPage ?? 1) + 1}`"
            >
                下一页
            </NuxtLink>
        </nav>
    </div>
</template>

Step 7: Run the Generator

kecare gen .

Kecare provides complete TypeScript type definitions to help you develop better:

KecareContext

Generator context, containing project path and other information:

type KecareContext = {
    projectPath: string;  // 项目根路径
}

ArticleVariant

Complete data for a single article:

type ArticleVariant = {
    title: string;           // 文章标题
    html: string;            // 渲染后的 HTML
    cover: string | undefined;   // 封面图片
    menu: string | undefined;    // 所属菜单
    desc: string;            // 文章描述
    lang: string;            // 语言代码（如 zh-CN）
    isOriginalLang: boolean; // 是否为原始语言
    hash: string;            // 文章唯一标识
    frontMatter: FrontMatter; // Front Matter 数据
    rawMarkdown: string;     // 原始 Markdown 内容
    [key: string | number | symbol]: any;  // 其他自定义字段
}

ArticlesRecord

Nested records of all articles:

type ArticlesRecord = Record<
    string,  // 文章 hash
    Record<
        string,  // 语言代码
        Omit<ArticleVariant, 'html'> & { 
            fsPath: string;   // 文件系统路径
            urlPath: string;  // URL 路径
        }
    >
>;

FrontMatter

Article metadata:

type FrontMatter = {
    menu: string;                 // 所属菜单
    cover: string | undefined;    // 封面图片
    layout: string | undefined;   // 布局模板
    title: string;                // 标题
    desc: string | undefined;     // 描述
    tags: Array<string>;          // 标签列表
    translate: Array<string>;     // 翻译语言列表
};

ModuleArticleTS

Article Detail Page Generator Type:

type ModuleArticleTS = {
    generator: (context: KecareContext, article: ArticleVariant) => Promise<{
        urlPath: string;    // 页面 URL
        fsPath: string;     // 文件路径
        template: string;   // 页面模板
    }>;
}

ModuleListTS

Article List Page Generator Types:

type ModuleListTS = {
    generator: (context: KecareContext, articles: ArticlesRecord) => Promise<
        Array<{ fsPath: string; template: string }> |
        { fsPath: string; template: string }
    >;
}

</script>

Frequently Asked Questions

Q: Where are the generated page files?

A: The location of the page file is determined by the fsPath you set in the generator. It is generally recommended to place it in the pages directory of the project.

Q: How to customize article URLs?

A: Modify the return value of urlPath in the article detail generator:

urlPath: ['blog', article.lang, article.hash].join('/')
// 结果: /blog/zh-CN/abc123

Q: How to get the original Markdown content of an article?

A: The rawMarkdown field in the ArticleVariant type contains the original Markdown content:

const rawContent = article.rawMarkdown;

Q: Why is it necessary to call kecareSDK.mounted()?

A: Kecare SDK provides Markdown extension features (such as code highlighting, Tabs, copy buttons, etc.). Calling the mounted() method initializes these features when the page is mounted.

Q: How to support multiple frameworks?

A: Kecare is framework-agnostic; you just need to return the corresponding framework's code in the template:

// React 示例
template: `
    import React from 'react';
    export default function Article({ article }) {
        return (
            <div>
                <h1>{article.title}</h1>
                <div dangerouslySetInnerHTML={{ __html: article.html }} />
            </div>
        );
    }
`

Q: How is the article ID generated?

A: The article ID (hash) is typically generated from the article's filename or content hash, ensuring each article has a unique and stable identifier.
