Kecareについて：はじめに

💡 はじめに：ドキュメントの中で不明確な点があれば、Issue や PR を提出して改善にご協力ください。あなたの貢献に心より感謝いたします！

Kecareは、現代的な静的ブログとノートサイト向けのジェネレーターフレームワークです。その最大の目標は「もう一つのHexoを作ること」ではなく、コンテンツシステムをテーマ/フレームワークから切り離し、ブログ機能を「プラグイン可能なモジュール」にすることです。

コアアドバンテージ：非侵入型設計

多くの従来のブログソリューション（例：Hexo）は、テーマ、ビルド、プラグイン、ディレクトリ構造などがそのシステムに固定されており、技術スタックを変更するには一からやり直す必要があります。

Kecare のソリューション

Kecareの核心的な強みは、**非侵襲性（Non-intrusive）**にあります。

一言で説明：もしあなたがフロントエンド開発者なら、既存のアプリケーションに「追加」でブログシステムを組み込むことができ、元のプロジェクトがどの技術スタックを使用しているかをほとんど気にする必要はありません。

プロジェクトに .kecare/ ディレクトリを作成し、指定の内容とルールに従って記述するだけで、Kecareはあなたが定義したルールに基づいてブログ関連のページとデータを生成します。

サポートされている技術スタック

あなたの元のプロジェクトは次のようなものでした：

タイプ フレームワーク例
Vue エコシステム Vue 3、Nuxt
React エコシステム React、Next.js
Svelte エコシステム Svelte、SvelteKit
従来型 SSR 任意のサーバーサイドレンダリングフレームワーク
レガシープロジェクト 生成結果を統合できれば何でも可

Kecareは、移行を強制せず、アーキテクチャを変更せず、ページの書き直しを要求しません。 それは「外部コンテンツ生産ライン」のように、ブログシステムを最も軽量な方法であなたのプロジェクトにマウントします。

二つの使用方法

Kecareは2つの「親しみやすい入門パス」を提供しており、ご自身のニーズに合わせて選択できます：

方法1：デフォルトテーマ（すぐに使える）

もしすぐに実行可能で美しく、執筆可能なブログを手に入れたいなら、公式が提供するデフォルトテーマを直接使用できます。

特徴 説明
技術スタック Nuxt + Vue 3
対象ユーザー 素早く執筆を始め、まずはブログを立ち上げたいユーザー
使用難易度 低い、Kecareの内部詳細をほとんど知る必要はありません
起動方法 ドキュメントに従って .kecare/ の内容を配置し、コマンドを実行するだけです

方法2：カスタムテーマ（真の非侵入型機能）

もし「技術スタックに依存しない」自由度をより重視するなら、カスタムテーマの道があなたにぴったりです。

あなたは：

  1. 任意の技術スタック/フレームワークで .kecare/ ディレクトリを新規作成する
  2. 好みの方法で生成ルールを定義する（例：記事をHTMLに変換する方法、ルーティングデータを生成する方法、メニューデータを生成する方法など）
  3. Kecareにプロジェクトが利用可能な成果物を出力させる（例：構造化データ、静的ページフラグメント、リソースインデックスなど）

そして、あなたのプロジェクトはたった一つのことだけをすればいい：

Kecare の成果物を接続してレンダリング（または直接公開）すれば、一連の「外部ブログシステム」が完成します。

まとめ

Kecareのデザイン理念：

  • ブログ機能は追加機能です：既存プロジェクトのリファクタリングは不要
  • 移植性が高い：コンテンツとテーマが分離されている
  • テーマに依存しない：任意の技術スタックに対応

これがKecareの理想的な姿です：ブログシステムをあなたのプロジェクトの「外部モジュール」にし、「すべてを書き直す」理由にしないことです。
