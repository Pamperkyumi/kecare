# 什么是Kecare

**面向多语言内容站的现代静态内容生成器**

Kecare 不只是把 Markdown 渲染成网页。
它把内容编译、文章翻译、页面生成和阅读体验放进了一条完整流程里，让你用更少的重复工作，做出更灵活、更现代的内容站。

如果你想要的不是“再搭一个传统博客”，而是一个更适合 Vue / Nuxt 生态、同时又能兼顾多语言与内容组织的方案，Kecare 就是为这件事准备的。
## Kecare 解决的是什么问题？

很多博客生成器都能把文章生成页面，但当你继续往前走，问题就会开始出现：

- 多语言内容维护成本高
- 主题结构固定，改起来很重
- 内容、模板、搜索、目录、代码高亮往往分散在不同插件里
- 想接入 Vue / Nuxt 主题时，经常需要额外改造

Kecare 的思路很直接：

**先把内容整理成可处理的数据，再按模块生成页面，最后用官方 SDK 补齐阅读体验。**

这让它不只是一个“写文章的工具”，更像一个适合内容站长期演进的生成方案。

## 为什么选择 Kecare？

### 1. 多语言不是附加功能，而是主流程的一部分

Kecare 内置 AI 翻译能力。你只需要在 Front Matter 中声明目标语言，它就可以按文章生成对应语言版本。

```yaml
---
title: 我的第一篇文章
translate: [zh-CN, en-US, ja-JP]
---
```

它不是简单把整篇文章丢给模型，而是把翻译放进内容处理流程里：

- 保留代码块和 Markdown 结构
- 支持按段落缓存，避免重复翻译
- 使用兼容 OpenAI API 的接口方案，便于接入兼容 API

对于需要持续更新、多语言同步发布的博客和内容站，这一点非常实用。

## 2. 模板不是一整套主题，而是可拆分的模块

传统博客工具通常围绕“主题”组织页面。主题一旦变复杂，改某一个页面，往往要牵动很多文件。

Kecare 采用的是更清晰的模块化方式：

| 模块               | 作用             |
| ------------------ | ---------------- |
| `*.article.ts`     | 生成文章详情页   |
| `*.list.ts`        | 生成列表页和分页 |
| `*.archives.ts`    | 生成归档页       |
| `*.menu.source.ts` | 生成导航菜单     |
| `search.ts`        | 生成搜索数据     |
| ...                | ...              |

这种方式的好处很直接：

- 页面职责更清楚
- 不同页面可以独立演进
- 更适合定制博客、专题页、归档页和内容聚合页
- 不容易陷入“主题越改越乱”的局面

## 3. 生成器之外，还有官方 SDK 补齐阅读体验

Kecare 不只关心“能不能生成页面”，也关心“页面出来以后好不好用”。

官方 SDK 已经覆盖了一批内容站常见能力：

- 自动目录导航
- 代码高亮
- 代码块复制
- Markdown 原文复制
- Tabs 交互
- 数学公式支持
- 配套样式注入

这意味着你不需要再从零拼装一堆前端小功能，生成器和阅读体验可以用同一套思路接起来。

### 4. 基于 Bun，整个流程更轻更直接

Kecare 使用 Bun + TypeScript 构建。

Bun 官方将自己定位为一套面向 JavaScript / TypeScript 的完整工具链，包含运行时、包管理器、测试器和 bundler，并强调更快的启动与执行体验。

对 Kecare 来说，这带来的价值不是一句“更快”就能概括，而是：

- CLI 启动更轻
- TypeScript 使用更直接
- 文件扫描、构建、写入链路更平滑
- 更适合作为现代内容生成器的底层运行时

## Kecare 和 Hexo、Hugo、VitePress 到底差在哪？

这几类工具都能做内容站，但路线并不一样。

| 维度            | Hexo                   | Hugo                | VitePress            | Kecare                             |
| --------------- | ---------------------- | ------------------- | -------------------- | ---------------------------------- |
| 内容组织方式    | 博客框架 + 主题/插件   | 内容目录 + 内建能力 | Markdown 文件即页面  | 内容先编译，再按模块生成页面       |
| 多语言          | 主要依赖插件或额外方案 | 官方内建            | 官方内建             | 文章级多语言变体 + AI 翻译         |
| 模板方式        | 主题机制               | Go Template         | Vue 主题             | 模块化模板                         |
| Vue / Nuxt 结合 | 一般                   | 弱                  | 很强                 | 很强                               |
| 搜索            | 常见做法是插件扩展     | 常见做法是外接方案  | 默认主题支持本地搜索 | 可生成搜索索引与文章数据           |
| 阅读增强        | 依赖主题或插件         | 内建部分能力        | 官方默认体验较完整   | 官方 SDK 提供一整套增强能力        |
| 适合的内容形态  | 传统博客               | 通用内容站          | 文档站               | 博客、知识库、专题页、多语言内容站 |
