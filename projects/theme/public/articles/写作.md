# 写作

在 Kecare 中，文章有固定的存放位置和格式规范。本文档将教你如何创建和编写文章。

---

## 文章存放位置

在 Kecare 中，文章统一存放在以下目录：

| 项目 | 说明 |
|------|------|
| 目录位置 | `主题目录/.kecare/articles/` |
| 文件类型 | 目前仅支持 `*.md`（Markdown 文件） |
| 命名建议 | 使用稳定、可读的英文文件名 |

### 命名规范

推荐使用以下命名方式：

```txt
getting-started.md     # 推荐：小写字母 + 连字符
my-first-post.md       # 推荐：清晰易懂
hello-kecare.md        # 推荐：描述性命名
```

> ⚠️ **注意**：避免频繁更改文件名。文章的 ID 通常由文件名哈希生成，文件名变化可能导致旧链接失效。

---

## 创建文章

### 步骤一：新建文件

在 `.kecare/articles/` 目录下创建一个新的 Markdown 文件：

```bash
# 示例：创建一篇新文章
touch .kecare/articles/hello-kecare.md
```

### 步骤二：编写内容

打开文件，开始编写你的文章内容。

---

## 文章结构

一篇完整的文章通常包含两部分：**Front Matter** 和 **正文内容**。

### 完整示例

```markdown
---
title: Hello Kecare
date: 2026-01-22 12:00:00
tags:
  - kecare
  - blog
desc: 这是我的第一篇 Kecare 文章
author: Pamper
menu: test
translate: ['zh-CN', 'en-US', 'ja-JP']
---

# Hello Kecare

这是一段正文内容。

## 第一章

- 支持 Markdown 标题
- 支持列表
- 支持代码块

\`\`\`ts
console.log("Hello, Kecare!");
\`\`\`

## 第二章

更多内容...
```

## 运行生成器

写完文章后，需要运行生成器才能在浏览器中看到更新。
