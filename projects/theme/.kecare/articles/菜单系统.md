# 菜单系统

如果你使用博客来写文档（比如你现在读的这一页），通常左侧会有一个菜单栏/目录栏帮助跳转。Kecare 提供了一套简单、统一的菜单编写方式。

---

## 菜单存放位置

在 Kecare 中，菜单有固定的存放位置：

| 项目 | 说明 |
|------|------|
| 目录位置 | `主题目录/.kecare/menus/` |
| 文件命名 | `*.menu.source.ts` |

> ⚠️ **注意**：只有放在这个目录下，Kecare 才能扫描到并生成对应的菜单。

---

## 创建菜单文件

### 文件命名规则

菜单文件必须遵循以下命名格式：

```txt
<菜单名称>.menu.source.ts
```

示例：

```txt
.kecare/menus/
├── kecare-docs.menu.source.ts    # Kecare 文档菜单
├── blog.menu.source.ts           # 博客菜单
└── api.menu.source.ts            # API 文档菜单
```

> 💡 **提示**：文件名前缀（如 `kecare-docs`）将成为你在文章 Front Matter 中引用的 `menu` 值。

## 编写菜单内容

### 基本结构

在菜单文件中导出 `navItems` 数组：

```ts
// .kecare/menus/kecare-docs.menu.source.ts
import type { NavItem } from "kecare";

export const navItems: NavItem[] = [
    {
        text: "分组名称",
        level: 1,
        items: [
            {
                text: "菜单项",
                link: "./文章路径",
                level: 2,
            },
        ],
    },
];
```

## 在文章中启用菜单

### 步骤一：在 Front Matter 中添加 menu 字段

在需要显示菜单的文章 Markdown 文件的 Front Matter 中添加 `menu` 字段：

```yaml
---
title: 快速开始
menu: kecare-docs    # 菜单 key
---
```

### 步骤二：确保菜单文件存在

`menu` 的值对应菜单文件名的前缀：

| menu 值 | 对应的菜单文件 |
|---------|----------------|
| `kecare-docs` | `.kecare/menus/kecare-docs.menu.source.ts` |
| `blog` | `.kecare/menus/blog.menu.source.ts` |
| `api` | `.kecare/menus/api.menu.source.ts` |

### 规则总结

```
menu 值 = .menu.source.ts 的文件名前缀
```

## NavItem 结构说明

### 类型定义

Kecare 的 `NavItem` 支持两种节点类型：

```ts
type NavItem =
    | { text: string; link: string; level: number; desc?: string; icon?: string }  // 叶子节点
    | { text: string; items: NavItem[]; level: number };                           // 分组节点
```

### 叶子节点

可点击跳转的菜单项：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `text` | `string` | 是 | 显示文本 |
| `link` | `string` | 是 | 跳转链接 |
| `level` | `number` | 是 | 层级深度 |
| `desc` | `string` | 否 | 描述文本（可选） |
| `icon` | `string` | 否 | 图标名称（可选） |

### 分组节点

仅作为分类，不可点击：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `text` | `string` | 是 | 分组名称 |
| `items` | `NavItem[]` | 是 | 子菜单项 |
| `level` | `number` | 是 | 层级深度 |

### link 字段说明

`link` 指的是要跳转到的文章路径：

```ts
{
    text: "快速开始",
    link: "./快速开始",    // 相对路径
    level: 2,
}
```

路径规则：
- 使用相对路径格式：`./文章名称`
- 对应文章的路由或 slug
- 不需要文件扩展名

## 常见问题

### Q: 一个文章能绑定多个菜单吗？

A: 通常一个文章只绑定一个 `menu`。如果需要多个菜单，需要主题侧支持（例如合并渲染）。

### Q: 菜单文件必须叫 `*.menu.source.ts` 吗？

A: 是的。这是 Kecare 扫描菜单文件的约定命名，不按规则命名可能无法被识别。

### Q: 如何添加菜单图标？

A: 使用 `icon` 字段：

```ts
{
    text: "快速开始",
    link: "./快速开始",
    level: 2,
    icon: "rocket",    // 图标名称
}
```

> 💡 **提示**：图标支持取决于主题实现。
