# 关于 Kecare：从这里开始

> 💡 **写在前面**：如果你觉得文档中有不清楚的地方，欢迎提 Issue 或 PR 帮助改进，非常感谢你的贡献！

Kecare 是一个面向现代静态博客与笔记站点的生成器框架。它最大的目标不是"再做一个 Hexo"，而是把**内容系统**从**主题/框架**中解耦出来，让你的博客能力变成"可插拔模块"。

---

## 核心优势：非侵入式设计

很多传统博客方案（如 Hexo）会把你带进一套固定的生态里：主题、构建、插件、目录结构等都与其绑定，想要更换技术栈就得重来。

### Kecare 的解决方案

Kecare 的核心优势在于**非侵入性（Non-intrusive）**。

> **一句话解释**：如果你是前端开发者，你可以直接给你的现有应用"加装"一套博客系统，几乎不需要关心原项目使用的是什么技术栈。

你只需要在项目中新建一个 `.kecare/` 目录，并按约定写好内容与规则，Kecare 就会根据你定义的规则生成博客相关的页面和数据。

### 支持的技术栈

你原本的项目可以是：

| 类型 | 框架示例 |
|------|----------|
| Vue 生态 | Vue 3、Nuxt |
| React 生态 | React、Next.js |
| Svelte 生态 | Svelte、SvelteKit |
| 传统 SSR | 任何服务端渲染框架 |
| 老项目 | 只要你能接入生成结果即可 |

**Kecare 不强迫你迁移，不改变你的架构，不要求你重写页面。** 它像一个"外置的内容生产线"，把博客系统以最轻的方式挂载到你的项目中。

---

## 两种使用方式

Kecare 提供了两条"友好上手路径"，你可以根据自己的需求选择：

### 方式一：默认主题（开箱即用）

如果你只想立刻拥有一个能运行、美观、可写作的博客，可以直接使用官方提供的**默认主题**。

| 特性 | 说明 |
|------|------|
| 技术栈 | Nuxt + Vue 3 |
| 适合人群 | 想快速开始写作、先把博客运行起来的用户 |
| 使用难度 | 低，几乎不需要了解 Kecare 内部细节 |
| 启动方式 | 按文档放好 `.kecare/` 内容并运行命令即可 |

### 方式二：自定义主题（真正的非侵入式能力）

如果你更在意"与技术栈无关"的自由度，自定义主题路线就是为你准备的。

你可以：

1. **在任何技术栈/框架中**新增一个 `.kecare/` 目录
2. **用你喜欢的方式定义生成规则**（如文章如何转 HTML、如何生成路由数据、如何生成菜单数据等）
3. **让 Kecare 输出你项目可消费的内容产物**（如结构化数据、静态页面片段、资源索引等）

然后你的项目只需要做一件事：

> **把 Kecare 的产物接进来渲染（或者直接发布），就完成了一套"外挂式博客系统"。**

## 总结

Kecare 的设计理念：

- **博客能力是附加的**：不需要重构现有项目
- **可移植的**：内容与主题解耦
- **与主题不绑定的**：支持任意技术栈

这就是 Kecare 的理想状态：让博客系统成为你项目的"外挂模块"，而不是"全部重写"的理由。
